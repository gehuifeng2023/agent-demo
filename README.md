# Agent Demo

一个使用Go语言构建的代理演示项目。

## 项目概述

`agent-demo` 是一个 Go 项目，用于演示和实现代理相关的功能。项目遵循清晰的代码组织结构和开发规范。

## 技术栈

- **语言**: Go 1.25
- **依赖**:
  - `gopkg.in/yaml.v3 v3.0.1` - YAML配置文件处理

## 项目结构

```
agent-demo/
├── cmd/                    # 命令行应用入口
├── internal/               # 内部包（私有实现）
├── docs/                   # 项目文档
├── go.mod                  # Go模块定义
├── go.sum                  # Go依赖校验
├── AGENTS.md              # 开发指南和规范
└── .gitignore             # Git忽略文件
```

## 开发指南

本项目遵循严格的开发规范，详见 [AGENTS.md](./AGENTS.md)。

### 核心原则

- **增量开发优于大规模重构** - 小的、可编译和通过测试的改动
- **学习现有代码** - 在实现前研究和规划
- **实用胜过教条** - 适应项目现实情况
- **意图清晰优于复杂技巧** - 保持代码简洁易懂

### 开发流程

1. **理解** - 研究现有代码中的模式
2. **测试** - 先写测试（红）
3. **实现** - 编写最小代码使测试通过（绿）
4. **重构** - 在测试通过的基础上清理代码
5. **提交** - 清晰的提交信息链接到计划

### 代码质量标准

每次提交必须满足：
- ✅ 代码能成功编译
- ✅ 所有现有测试通过
- ✅ 新功能包含相应测试
- ✅ 遵循项目格式和linting规范

### 定义完成

- [ ] 测试编写完成且通过
- [ ] 代码遵循项目规范
- [ ] 无linter/formatter警告
- [ ] 提交信息清晰
- [ ] 实现与计划一致
- [ ] 无无号的TODO注释

## 快速开始

### 前提条件

- Go 1.25 或更高版本

### 安装依赖

```bash
go mod download
```

### 构建

```bash
go build ./cmd/...
```

### 测试

```bash
go test ./...
```

## 注意事项

### 禁止事项

- ❌ 使用 `--no-verify` 绕过提交钩子
- ❌ 禁用测试而不修复它们
- ❌ 提交无法编译的代码
- ❌ 做假设 - 用现有代码验证

### 必须做的事

- ✅ 增量提交可工作代码
- ✅ 开发过程中更新计划文档
- ✅ 从现有实现中学习
- ✅ 多次失败后停下来重新评估（最多3次尝试）

## 架构原则

- **组合优于继承** - 使用依赖注入
- **接口优于单例** - 启用测试和灵活性
- **显式优于隐式** - 清晰的数据流和依赖关系
- **尽可能进行测试驱动开发** - 从不禁用测试，修复它们

## 错误处理

- 快速失败，提供描述性信息
- 包含调试上下文
- 在适当级别处理错误
- 永不静默吞掉异常
