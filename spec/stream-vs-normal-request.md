# 普通请求与流式请求

## 一、核心区别

普通请求和流式请求的主要区别，不是模型的生成能力不同，而是模型结果的返回方式不同。

| 对比项 | 普通请求 | 流式请求 |
| --- | --- | --- |
| 请求参数 | `stream: false`，或不传该参数 | `stream: true` |
| 返回时机 | 模型生成完整答案后一次返回 | 模型生成一小段就返回一小段 |
| 响应类型 | 普通 JSON | SSE：`text/event-stream` |
| 客户端处理 | 等待完整响应后解析 | 持续读取并拼接增量内容 |
| 用户体验 | 首字出现较慢 | 可以较早看到模型开始回答 |
| 连接生命周期 | 返回完成后关闭 | 直到生成结束、失败或取消才关闭 |
| 错误处理 | 通常直接返回 HTTP 错误 | 可能已经收到部分内容后才收到错误 |
| 适合场景 | 结构化结果、后台任务、短回答 | 聊天、长回答、实时展示 |

流式请求并不意味着模型一定更快完成，只是让客户端更早看到部分结果。总生成时间可能相近，但首字响应时间通常更短。

## 二、当前项目的调用链

### 普通请求

```text
POST /api/v1/chat
    ↓
llmClient.Generate()
    ↓
等待完整答案
    ↓
返回一个 JSON
```

### 流式请求

```text
POST /api/v1/chat/stream
    ↓
llmClient.Stream()
    ↓
DeepSeek 设置 stream: true
    ↓
持续读取 SSE 数据块
    ↓
持续转发给客户端
```

## 三、DeepSeek 流式响应

DeepSeek 的流式接口会返回一系列 SSE 数据块，示例：

```text
data: {"choices":[{"delta":{"content":"你好"}}]}

data: {"choices":[{"delta":{"content":"，这是"}}]}

data: {"choices":[{"delta":{"content":"流式输出"}}]}

data: [DONE]
```

客户端需要按顺序拼接每个数据块中的 `delta.content`：

```text
你好 + ，这是 + 流式输出
= 你好，这是流式输出
```

其中 `data: [DONE]` 表示模型已经完成生成，客户端可以停止读取。

## 四、如何理解“流”

可以把流式请求理解成一根水管：

- 普通请求：等水桶装满后一次交付
- 流式请求：水从水管中持续流出，客户端边接收边使用
- `data: [DONE]`：水管已经关闭，内容发送完毕
- 客户端断开：相当于关闭水龙头，服务端应取消模型请求

因此，流式请求拿到的不是一个完整答案，而是一组按顺序到达的答案片段。

## 五、请求选择规则

可以按下面的规则选择请求方式：

```text
需要完整结果、后续要解析 JSON
    → 使用普通请求

需要用户实时看到文字
    → 使用流式请求
```

更具体地说：

- 需要结构化 JSON、数据库写入或后台批处理时，使用普通请求。
- 需要聊天窗口逐字展示答案时，使用流式请求。
- 需要判断整个回答是否成功时，使用普通请求，或者等待流式请求的结束事件。
- 使用流式请求时，必须处理部分内容、连接中断、上下文取消和模型错误。

## 六、请求示例

### 普通请求

```bash
curl -X POST http://localhost:8080/api/v1/chat \
  -H 'Content-Type: application/json' \
  -d '{"question":"解释一下 RAG"}'
```

普通请求会等待完整结果，最后返回一个 JSON 响应。

### 流式请求

```bash
curl -N -X POST http://localhost:8080/api/v1/chat/stream \
  -H 'Content-Type: application/json' \
  -H 'Accept: text/event-stream' \
  -d '{"question":"解释一下 RAG"}'
```

`curl -N` 用于关闭客户端输出缓冲，否则已经收到的数据可能不会立即显示。

当前项目的流式接口会依次返回：

1. `event: meta`：会话 ID、回答类型和知识来源。
2. 多个 `data` 事件：模型返回的文本增量。
3. `data: [DONE]`：流式响应成功结束。
4. `event: error`：发生错误时返回错误信息。

## 七、一句话总结

普通请求返回“最终答案”，流式请求返回“答案片段序列”；前者适合程序处理完整结果，后者适合实时展示生成过程。
